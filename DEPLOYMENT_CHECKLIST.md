# 📋 部署检查清单

在开始部署前，请确保完成以下检查项。

## ✅ 部署前检查

### 代码准备
- [ ] 已创建 GitHub 账号
- [ ] 已安装 Git
- [ ] 已运行 `init-git.bat` 初始化仓库
- [ ] 代码已推送到 GitHub

### 环境变量
- [ ] 已获取心流开放平台 API Key（https://platform.iflow.cn）
- [ ] 了解 API Base URL（默认：https://apis.iflow.cn/v1）

### 平台账号
- [ ] 已注册 Railway 账号（https://railway.app）
- [ ] 已注册 Vercel 账号（https://vercel.com）
- [ ] GitHub 已授权给 Railway 和 Vercel

---

## 🚀 后端部署检查（Railway）

### 基础配置
- [ ] 在 Railway 创建了新项目
- [ ] 选择了正确的 GitHub 仓库
- [ ] Root Directory 设置为 `backend`

### 环境变量
- [ ] 设置 `NODE_ENV=production`
- [ ] 设置 `PORT=3001`（Railway 通常会自动设置）

### 部署验证
- [ ] 部署状态显示 SUCCESS
- [ ] 访问 `/health` 端点返回 `{"status":"ok"}`
- [ ] 复制后端 URL（如：https://xxx.up.railway.app）

---

## 🎨 前端部署检查（Vercel）

### 基础配置
- [ ] 在 Vercel 导入了 GitHub 项目
- [ ] Root Directory 设置为 `frontend`
- [ ] Framework Preset 选择 Create React App
- [ ] Build Command: `npm run build`
- [ ] Output Directory: `build`

### 环境变量
- [ ] 设置 `REACT_APP_API_URL` 为 Railway 后端 URL

### 部署验证
- [ ] 部署状态显示 Ready
- [ ] 访问 Vercel 提供的域名可以打开网站
- [ ] 能够正常配置 API Key
- [ ] 能够上传文献并获取分析结果

## 🆘 故障排查

### 常见问题速查

**前端无法连接后端**
- 检查 `REACT_APP_API_URL` 环境变量
- 检查后端服务是否运行
- 查看浏览器控制台错误信息

**后端部署失败**
- 查看 Railway Deploy Logs
- 检查 `package.json` 配置
- 确认 Node.js 版本兼容性

**文件上传失败**
- 检查文件大小是否超限
- 查看后端日志
- 检查磁盘空间

**API 调用失败**
- 验证 API Key 是否正确
- 检查网络连接
- 查看后端日志
